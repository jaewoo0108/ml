## 1. CART (Classification And Regression Trees)

Decision Tree: root, decision, leaf 노드로 구성되어 있는 트리 구조.

데이터가 주어졌을 때, 다양한 의사결정 트리들을 만들 수 있지만 어떤 트리들은 훈련 데이터셋에, 또 어떤 트리들은 테스트 데이터셋에 좋은 성능을 낸다.

→ 훈련 데이터셋에서도 좋은 성능을 내면서 새로운 데이터에 대해서도 일반화가 잘되는 트리 선택.

Tree가 커지면 커질수록 overfitting(과적합)의 위험이 커지게 되고 다루기 힘들어진다.

## 2. 분류 트리 (Classification Tree)

분류에 사용되는 결정 트리.

### <결정트리를 만드는 과정에서 내리는 몇가지 중요한 결정들>

- 각 노드에서 데이터를 나누기 위해 어떤 특성을 사용할지 어떻게 선택하는가?: 정답의 순도(purity)를 최대화하는 특성을 사용한다.
- 데이터를 나누는 분할 작업을 언제 멈출 것인가?
  1. 순도가 100% 도달시 멈춤
  2. 트리의 최대 깊이 제한 - 우리가 직접 설정하는 파라미터
  3. 순도의 개선치가 너무 작을때 - 과적합 방지
  4. 노드 내 데이터의 개수가 너무 적을 때 - 과적합 방지

### <엔트로피 함수 (Entropy function)>

데이터셋의 불순도(impurity)를 측정하는 함수 (0~1 사이의 값을 출력). 불순도가 높을수록 1에 가까워지고 낮을수록 0에 가까운 값을 출력한다.

관례적으로 H()로 엔트로피 함수를 표기한다. (이진 분류일때의 엔트로피 함수 공식)

$$H(p) = -p \log_2(p) - (1-p) \log_2(1-p)$$

엔트로피 함수를 이용해서 각 노드에서 어떤 특성을 이용해 데이터를 나눌지 결정할 때, 데이터 분할의 결과로 나온 왼쪽 branch와 오른쪽 branch에서 나온 엔트로피 값을 단순히 하나씩 비교하기보단 하나의 값으로 결합해서 비교하는게 편리하다.

하나의 값으로 두 엔트로피 값을 합칠 때, 단순히 두 값의 평균을 구하는 방식은 데이터의 개수를 고려하지 않기 때문에 가중평균을 이용해 구해야한다. (데이터 개수를 고려하지 않는 단순 평균은 전체 데이터의 대다수를 차지하는 큰 그룹의 불순도와 소수에 불과한 작은 그룹의 불순도를 동등하게 취급하기 때문에 실제 분할의 성공 여부를 왜곡한다.)

### <정보 이득 (Information Gain)>

하지만 이렇게 구한 가중평균 엔트로피 값만으로 어떤 특성을 분할에 사용할지 결정하지 않는다. 단순히 가중평균 엔트로피를 계산하는 대신 "아예 분할하지 않았을 때와 비교해서 엔트로피가 얼마나 감소했는지"를 계산한다. → Information gain(정보 이득): 엔트로피의 감소량.

$$IG = H(p_{root}) - \left( w_{left} H(p_{left}) + w_{right} H(p_{right}) \right)$$

- $H(p_{root})$: 질문을 던지기 전 부모 방의 원래 엔트로피
- $w_{left} H(p_{left}) + w_{right} H(p_{right})$: 질문을 던진 후 나누어진 좌우 방의 가중 평균 엔트로피

→ 상위 노드의 엔트로피에서 가중 평균 엔트로피 공식을 뺀 값.

굳이 엔트로피 감소량(Information Gain)을 계산하는 이유: 어떤 질문이 좋은지 비교할 때는 두 방식의 결과 똑같지만 IG는 언제 분할을 멈출지 결정하는 정지조건 중 하나로 쓰이기 때문에, 엔트로피의 감소량이 임계값보다 작다면 굳이 분할해서 트리의 크기를 키우고 과적합 위험을 높일 필요가 없다.

## 3. 결정 트리 생성 과정 (Decision Tree building Process)

1. root 노드에서 모든 입력 특성들로 데이터 분할을 시작해, 모든 입력 특성에 대해 Information gain을 구해서 가장 높은 IG값을 가지는 특성을 분할 특성으로 선택한다.
2. 선택한 특성 기준에 따라서 데이터셋을 각 데이터가 가진 해당 특성의 값을 이용해 두개의 subset로 분할한다.
3. 나눠진 두개의 subset에서도 정지조건이 충족될 때까지 분할 과정을 계속해서 반복한다. (정지조건들: 노드가 순도가 100%가 되었거나, 최대 깊이 초과할 때, IG가 임계값보다 작을 때, 데이터 개수가 적을 때)
4. 정지조건이 충족된 경우, 해당 노드를 최종 예측을 하는 leaf 노드로 만든다.

→ 의사 결정 트리: 작은 하위 트리들의 합으로 표현되며 Recursive algorithm 원리를 가진다. 루트 노드에서 전체 의사결정 트리를 만드는 방법이 하위 가지에서 더 작은 의사결정 트리를 만드는 방식과 완전히 똑같다는 것을 의미.

## 4. 범주형 / 연속형 특성 다루기

결정 트리에서 Categorical features와 Continuous features를 다루는 방법.

### <One-hot encoding>

범주형 특성이 k개의 가능한 값을 가질 수 있다면, 이를 0 또는 1의 값만 가질 수 있는 k개의 이진 특성을 생성하여 대체하는 기법 (이진 특성들 중 하나는 무조건 1의 값을 가진다).

→ 1과 0을 이용하여 categorical feature들을 encoding 해주기 때문에 다중 선택지로 인해서 트리의 분할 구조가 변경되는 문제가 발생하지 않는다.

### <연속형 특성>

연속형 특성을 노드에서 분할하려고 하는 경우엔, 정렬된 훈련 데이터들의 인접한 중점을 모두 임계값 후보로 설정한다. 각 임계값 후보를 기준으로 데이터를 분할한 후 Information gain 공식을 적용하여 가장 높은 IG값을 얻는 값을 임계값으로 설정한다.

## 5. 회귀 트리 (Regression Tree)

회귀에 사용되는 결정 트리.

숫자 값을 예측하는 결정트리를 만드는 과정에서의 key decision들은 classification tree와 마찬가지로 어떤 데이터 특성을 이용해 데이터를 나눌지 결정하는 것이다.

- 데이터들의 불순도인 entropy 값을 낮추려고 했던 반면에, Variance(분산)의 값을 가장 많이 낮춰주는 특성을 이용한다.
- 가중평균을 적용한 분산 값을 구한 후, 분류 결정 트리와 동일하게 최종적으로 Variance Reduction(분산 감소량)을 구해서 데이터 특성들을 비교한다.

$$VR = Var(y_{root}) - \left( w_{left} Var(y_{left}) + w_{right} Var(y_{right}) \right)$$

variance: 데이터들이 흩어져 있는 정도(숫자들이 얼마나 넓게 퍼져 있는가를 계산한 값).

→ 분할된 하위 서브트리들에 대해 동일한 분할 과정을 재귀적으로 반복하고, 정지 조건을 충족하는 경우에 분할을 멈추고 Leaf Node로 확정한다.

## 6. 트리 앙상블 (Tree Ensemble)

단일 결정 트리를 쓰면 훈련 데이터의 조그만한 변경에도 매우 민감해서 결과적으로 완전히 다른 모양의 결정트리가 생성된다는 문제가 있다.

결정 트리 모델을 데이터의 변경 또는 수정에 덜 민감하게 만들고, 더욱 견고하게 만드는 방법으로 여러 결정 트리를 만들어 함께 활용하는 방식인 트리 앙상블 기법을 사용한다 (전체 트리들의 다수결 투표(Bagging)를 통해 최종 정답 결정).

앙상블 기법에서 다양한 결정 트리를 만들 때, 트리의 수를 크게 설정하는 것이 모델의 성능을 낮추진 않지만 특정 지점을 넘어서면 투자 대비 효과 감소가 발생하기에 일반적으로 트리의 수를 너무 크게 설정하지 않는다.

### <복원 추출 (Sampling with replacement) / Bootstrapping>

복원 추출: 원본 데이터가 들어가 있는 데이터셋에서 데이터를 뽑고, 뽑은 데이터를 다시 넣고(복원) 뽑는 과정을 반복하는 것. → 중복 데이터와 누락 데이터가 발생한다.

트리 앙상블을 구축할 때 원본 훈련 세트와 조금씩 다른 여러개의 무작위한 훈련 세트를 만들기 위해서 적용되는 방식.

### <랜덤 포레스트 (Random forest algorithm) / Bagging>

Bootstrapping 절차를 사용하더라도, 데이터 내에 압도적으로 강력한 특성이 존재하는 등의 문제로 인해서 트리간의 상관관계가 높아진다는 문제가 발생하기에 각 노드에서 특성을 선택할때 무작위성을 더 추가하는 변화를 줘서 학습하는 트리들이 훨씬 더 달라지게 만듬으로써 정확하고 강력한 성능을 가질 수 있다.

특성의 부분집합을 매 노드마다 무작위로 선택해서 ($k = \sqrt{n}$ (전체 특성 개수의 제곱근)), 오직 그 부분집합 내에서만 IG 또는 Variance reduction 값을 비교함으로써 트리마다 완전히 다르고 다양한 구조를 가지게 된다.

### <XGBoost / Boosting>

부스팅: 이전 트리들이 뭉친 앙상블의 결과를 보고 아직 잘 맞히지 못하는 데이터를 확인한 후 다음 루프에서 트리를 만들 때 잘 맞히지 못했던 데이터들에 대해 더 많은 가중치를 주어서 또 다른 훈련 세트를 생성할 때, 뽑힐 확률을 더 높여주는 것 (앞선 트리의 결과를 보고 다음 트리들이 만들어지는 것).

XGBoost: 부스팅을 구현하는 알고리즘, 분할을 언제 멈출지 결정하는 정지기준도 기본값으로 설정, 과적합을 방지하기 위한 정규화 기능도 내장한다. (데이터를 매번 복원 추출하지 않고 새로 뽑는 대신, 훈련 데이터들 각각에 다른 가중치를 직접 부여하는 방식 이용하기에 연산 속도, 효율성 증가.)

### <트리 앙상블이 단일 결정 트리보다 압도적으로 강건한 이유>

→ 복원 추출 절차를 통해서 일부 데이터의 누락이나 중복을 통해 수많은 작은 데이터 변화들을 의도적으로 경험하고 여러 결정 트리의 예측을 평균내거나 다수결 투표등을 통해 평균화 했기 때문에 실전에서 새로운 노이즈나 변형이 추가되더라도 모델의 최종 예측 결과는 흔들리지 않고 안정적으로 유지된다.

## 7. 트리 앙상블 구현

1. 데이터셋 전처리: 범주형 특성 중에서 3개이상의 값을 가지는 범주형 특성은 one-hot encoding 통해서 두 개의 값만을 가지는 이진 변수로 변형하고, 데이터셋을 훈련 세트, 검증 세트, 테스트 세트로 나눈다.
2. Decision Tree, Random forest, XGBoost 중 어떤 알고리즘을 쓸지 결정한 후, 각 알고리즘마다 설정해야할 하이퍼 파라미터의(max_depth, min_samples_split, n_estimators, max_features 등등) 최적 값을 찾는 과정을 거친다.
3. 각 하이퍼 파라미터에 대해서 각각의 값을 시도했을 때, 훈련 세트의 정확도와 검증 세트에서의 성능을 비교함으로써 가장 좋은 하이퍼파라미터 값 설정 (이상적으로, 튜닝하고자 하는 모든 하이퍼파라미터들의 조합을 모든 가능한 조합을 전부 확인하는 것(GridSearch)이 좋다).

XGBoost의 흥미로운 특징 (조기종료): 모델을 훈련(fit)하는 동안 평가용(검증) 데이터셋을 `eval_set = [(X_train_eval, y_train_eval)]`와 같이 함께 입력 받아서 매 반복마다 평가용 데이터셋에 대한 비용(오차)를 실시간으로 측정한다.

그렇기에 `early_stopping_rounds` 메소드를 통해 설정한 특정 횟수(라운드)동안 오차가 더 이상 감소하지 않고 정체된다면 훈련은 조기종료된다, 가장 좋은 성능을 기록한 반복 횟수인 `best_iteration` 저장. → 조기종료함으로써 트리의 개수가 많아져서 발생하는 과적합 방지 & 연산, 학습 시간 절약.

## 8. 언제 결정 트리를 쓸까? (Decision Tree vs Neural Network)

### <Decision Tree & Tree ensemble>

- 데이터가 행과 열로 정리된 정형 데이터(Tabular data)로 구성되어 있는 경우, 뛰어난 성능을 가진다
- 훈련 속도가 매우 빠르다
- 작은 크기의 결정 트리는 사람이 시각화해서 해석할 수 있다

### <Neural Network>

- 정형 데이터를 포함한 모든 종류의 데이터에서 좋은 성능을 보이지만, 그 중에서도 (이미지, 텍스트, 오디오)와 같은 비정형 데이터는 가장 좋은 성능을 보인다
- 결정 트리에 비하면 훈련 속도가 느리다
- 전이 학습을 사용 가능 (훈련 데이터셋의 크기가 매우 작더라도, 훨씬 더 큰 대용량 데이터셋으로 사전학습을 수행한 뒤 전이학습을 사용할 수 있다)
