## 2. 활성화 함수 (Activation Functions)

ReLU 활성화함수: $g(z) = \max(0, z)$

- 음수가 들어오면 0으로 만들고, 양수가 들어오면 양수 값 그대로 출력한다.

우리는 신경망을 설계할 때 각 뉴런에 사용할 활성화 함수로 ReLU, sigmoid, linear 등등에서 활성화함수인 g(z)를 선택한다.

(linear activation function을 사용하는 경우에, 단순히 g(z) = z = w·x + b 이기에 활성화함수를 사용하지 않는다고도 할 수 있다.)

### 출력층을 위한 활성화 함수 선택

어떤 정답값(label Y)을 예측하려고 하는지에 따라 다르게 선택해야한다.

- 분류 문제인 경우, 정답은 0 또는 1이기때문에 시그모이드 활성화함수가 가장 좋은 선택이다 → 신경망이 정답이 1이거나 0일 확률을 예측하는것을 배운다.
- 회귀 문제인 경우, 정답이 연속적인 숫자(음수, 양수 등등)를 예측하는 경우에 linear 활성화함수가 가장 좋은 선택이다 → 신경망의 출력 또한 음수와 양수가 나올 수 있기에.
- 회귀 문제에서 정답값이 오직 양수값만 나올 수 있는 경우, 음수값은 0으로 만들고 양수값은 그대로 내보내는 ReLU 활성화 함수가 가장 좋은 선택이다.

### 은닉층을 위한 활성화 함수

초기에는 시그모이드함수를 주로 사용했엇지만 현재는 렐루함수가 가장 흔하게 선택되며 사용되는 함수이다.

#### 렐루함수를 사용하는 이유

1. 연산 효율성: 시그모이드는 지수 연산과 inverse 역수 계산을 해야하는 반면에 렐루는 단순히 max(0,z) 연산만 하면 되기에 연산 속도가 빠르고 효율적이다.
2. 비용 함수의 지형 문제: ReLU함수는 그래프의 한쪽 부분(왼쪽 음수)만 완전히 평평한 반면, 시그모이드 함수는 그래프의 왼쪽과 오른쪽에서 둘다 평평하다. 활성화 함수가 평평한 곳이 많아지면 비용 함수의 지형도 덩달아 평평해져 경사하강법의 최적화 속도가 매우 느려진다. (경사하강법의 대상은 비용함수이지만, 활성화 함수는 비용함수의 일부(내부 계산에 포함)이기에 영향을 받는다.)

### 역전파와 기울기 소실 문제

경사하강법을 수행할 때, 비용 함수에서 계산된 오차를 출력층에서부터 입력층 방향으로 거꾸로 전파(역전파)시키며 가중치를 수정한다. 이때 각 뉴런을 거칠 때마다 해당 뉴런의 활성화 함수 기울기(미분값)를 곱해주게 된다.

- 활성화 함수가 시그모이드인 경우: 층을 거칠 때마다 기껏해야 기울기의 최댓값이 0.25이고 양 끝에서는 0에 가깝게 수렴하는 시그모이드의 기울기가 계속 곱해진다. 아무리 큰 오차를 주어도 깊은 층을 통과할수록 결국 0에 가까운 아주 작은 값으로 소멸되어, 입력층에 가까운 앞쪽 뉴런들은 가중치 학습이 제대로 이루어지지 않는다.
- 활성화 함수가 렐루인 경우: 음수 영역에서는 기울기가 완벽히 0이 되기에 오차전달이 끊기지만, 양수 영역에서는 기울기가 항상 1이기 때문에 비용함수에서 구한 오차가 왜곡되지않고 신경망의 깊은 곳까지 경사하강법이 잘 작동하게 된다.

## 3. 비선형 활성화 함수가 필요한 이유 (linear를 쓰면 안 되는 이유)

선형 회귀 모델의 그래프보다 더 복잡한 것에 fit 할 수 없게 된다 = 현실의 복잡한 데이터(이미지, 음성 등)는 절대 단순한 직선 하나로 분류하거나 예측할 수 없다.

여러 layer를 가진 신경망을 설계하고 각 뉴런들의 활성화함수를 linear 활성화함수로 사용하는 경우에, 신경망의 최종 출력은 하나의 linear regression으로 표현되기에 신경망을 여러 층으로 쌓은 이유가 없게된다.

→ 아무리 신경망의 층을 깊게 설계해도 행위 자체가 무의미해지고 하나의 거대한 직선 공식 (y = wx + b)으로 압축되어버린다.

그렇기 때문에 일반적으로 신경망의 은닉층에는 선형 활성화함수를 사용하지 않는다. (직선 공식에 직선공식을 아무리 더하고 곱해봐도 결국 하나의 거대한 직선이다.)

### 인공신경망과 ReLU 함수

단순해 보이는 ReLU의 꺾임이 여러 뉴런에 걸쳐 쌓이고 합쳐지면서, 전체 모델이 직선을 벗어난 비선형 함수를 그릴 수 있게 되는 원리.

ReLU함수를 사용하면 단순한 직선이 아닌 복잡하게 꺽이는 선인 비선형을 그릴 수 있는 이유는 ReLU 뉴런 하나는 입력이 특정 임계점(편향)을 넘으면 켜지는 스위치 기능때문이다.

특정 임계점전까지는 죽어있는(0) 개별 뉴런의 ReLU함수들이, 특정 임계점을 넘어서 켜지게된다면 뉴런은 자신의 기울기(가중치가 결정)를 출력 함수에 더한다. 입력값이 변하면서 어떤 뉴런들이 켜져 있느냐의 조합이 바뀌고, 그 조합이 바뀌는 지점마다 함수가 꺾인다. 이렇게 만들어진 여러 직선 구간들이 이어 붙어 piecewise linear 함수가 되며, 조각이 충분히 많으면 어떤 곡선이든 근사할 수 있다(보편 근사). 층을 깊게 쌓으면 이 꺾기가 합성되어 같은 뉴런 수로도 훨씬 더 많은 조각을 만들 수 있다.

- piecewise linear 함수: 여러개의 직선 토막을 이어 붙여 만든 함수

## 4. 다중 분류 (Multiclass Classification)

분류 문제에서, target Y가 2개 이상의 값을 가질 수 있게 되는것.

### 소프트맥스 회귀 (Softmax regression)

target Y가 N개의 possible output value를 가진다면:

$$z_j = \vec{w}_j \cdot \vec{x} + b_j \quad j = 1, \dots, N$$

$$a_j = \frac{e^{z_j}}{\sum_{k=1}^{N} e^{z_k}} = P(y = j \mid \vec{x})$$

Aj는 입력 특징으로 X가 주어졌을 때, Target Y가 j일 확률을 모델이 예측한 값을 나타낸다.

또한 A1부터 Aj까지의 모든 값을 더한 sum은 항상 1이 된다.

- 소프트맥스 함수의 독특한 특징: 출력값을 얻기 위해서 활성화 함수에 각각의 원소별($z_1, z_2$)로 적용했던 ReLU, sigmoid와 같은 다른 활성화함수들과 달리 $a_1$ 하나를 구하는데에도 $z_1, z_2, z_3$ 부터 $z_{10}$ 까지의 모든 값이 다 쓰이는 함수, 출력 확률값(a)들은 각각 z의 모든값에 의존한다.

→ 모든 뉴런의 확률 합을 정확히 100%(1)로 맞춰준다.

### 소프트맥스 회귀의 Cost 함수 (= SparseCategoricalCrossentropy)

실제 정답값(y)이 j라고 한다면, loss는 $-\log a_j$ 가 되고 $a_j$ 가 1에 가까울수록(커질수록) loss는 0에 가까워진다.

$$
loss(a_1, \dots, a_N, y) =
\begin{cases}
-\log a_1 & \text{if } y = 1 \\
-\log a_2 & \text{if } y = 2 \\
\quad \vdots \\
-\log a_N & \text{if } y = N
\end{cases}
$$

m은 전체 훈련데이터의 개수, n은 정답레이블의 후보 개수일때의 비용함수 공식:

$$J(\mathbf{w}, b) = -\frac{1}{m} \left[ \sum_{i=1}^{m} \sum_{j=1}^{N} 1\{y^{(i)} == j\} \log \frac{e^{z_j^{(i)}}}{\sum_{k=1}^{N} e^{z_k^{(i)}}} \right]$$

### Softmax output을 가진 신경망

신경망의 output layer의 뉴런 개수는 우리가 분류하고싶은 target의 개수로 설정한다.

## 5. 수치적 반올림 오차 (Numerical Roundoff Errors)

컴퓨터의 메모리는 유한하기 때문에 $e^z$ 와 같이 숫자가 엄청나게 커지거나 너무 작아지는 경우가 자주 발생하는 공식을 사용하는 경우에 컴퓨터가 저장할 수 있는 한계를 넘어서 무한대가(overflow) 되거나 0으로 사라지는(underflow) 일이 빈번하게 발생한다. → 반올림 오차(Round-off error)

같은 연산을 하더라도 계산 방식이 달라짐에 따라서 반올림 오차가 더 많이 혹은 더 적게 발생할 수 있다. 그렇기 때문에 기존의 구현은 소프트맥스의 확률값 a를 따로 구한 후 손실함수에 넘겨줬었지만, 이 과정을 통째로 텐서플로우에게 넘겨주면 내부적으로 수학적 약분, 항 재배치를 통해 반올림 오차를 줄일 수 있다.

컴퓨터의 반올림 오차를 줄이기 위해 수식을 하나로 합친다.

```python
model.compile(loss=SparseCategoricalCrossentropy(from_logits=True))
```

- 출력층의 활성화 함수를 그냥 linear 함수로 설정, loss함수를 설정할 때 `from_logits=True`로 설정
- 텐서플로우가 내부적으로 z 값을 가지고 수학적 재배치를 수행하여 반올림 오차가 최소화된 가장 정확한 오차를 계산 (= 손실함수가 활성화함수 역할까지 독점해서 한번에 계산)
- (결합된 수식은 로그 성질에 의해 지수 연산이 약분되면서 지수함수 때문에 숫자가 무한대로 터지거나 소수점이 잘려 나가는 반올림 오차를 차단하여 계산 정확도를 극대화)
- logits: 기본적으로 선형 계산으로 나온 숫자 $z$ 를 의미

Output layer의 활성화 함수를 linear로 설정하기 때문에 코드의 가독성이 떨어진다는 단점이 있지만 텐서플로우 내부에서 수치적 반올림 오차가 훨씬 적게 발생하도록 만들어준다.

## 6. Multi-Class vs Multi-Label Classification

- Multi-Class는 정답 레이블의 후보가 여럿 → 답을 그 중 하나로 결정
- Multi-Label은 정답 레이블 자체가 여러개 이기때문에 각 레이블이 서로 독립적이다 (= 있다/없다를 레이블마다 따로따로 판단 하기에 시그모이드 함수 사용)

## 7. 고급 최적화 (Advanced Optimization)

경사하강법이 진행되는 상황에 따라, 어떤 상황엔 학습률이 크면 좋고 또 다른 상황엔 학습률이 작으면 좋겠다고 바라게 된다.

Adam algorithm: 학습률을 자동으로 조정한다. 전체 모델에 똑같이 적용되는 하나의 전역 학습률을 쓰지 않고 대신, 각 매개변수마다의 각기 다른 학습률을 사용한다.

- 경사하강법의 매 단계에서 학습률에 곱하는 각 파라미터의 기울기값을 토대로, 기울기값이 일관적으로 같은 방향을 가리킨다면 학습률을 증가시키고 기울기값이 진동하게 된다면 학습률을 감소시킨다.

## 8. 신경망의 추가 레이어 타입 (Additional Layer Types)

- Dense layer: 각 뉴런의 출력값은 이전 층의 모든 뉴런의 출력값을 입력값으로 받아들이는것
- Convolution Layer: 각 뉴런이 이전 층의 모든 뉴런의 출력값을 받아들이지 않고 일부만 받아들이고 보는 것
  - 장점
    - 각 뉴런이 모든 뉴런의 출력값을 고려하지 않기때문에 연산이 빠르다
    - 더 적은 훈련데이터 필요 (= 과적합을 방지하기 쉽다)

## 9. 역전파 (Back Propagation)

backprop은 derivative를 구하는 가장 효율적인 방법이다. 연쇄 법칙을 적용해서 중간에 계산된 미분값을 재사용하기 때문에 효율적이다.

예시 계산 그래프:

$$w \rightarrow c = wx \rightarrow a = c + b \rightarrow d = a - y \rightarrow J = \tfrac{1}{2}d^2$$

w가 최종 비용함수 J에 끼치는 영향인 $\frac{\partial J}{\partial w}$ 는 연쇄법칙으로 여러 국소 미분의 곱으로 표현된다. (w가 최종 J에 주는 영향을 알려면, w는 J에 곧장 닿지 않고 c, a, d 등등의 중간 길목을 거쳐서 J가 만들어지기 때문에 그 길목에 있는 요소들이 각각 바로 다음 요소에 주는 영향력까지 전부 알아야한다.)

이때 J쪽으로부터 미분값을 구해 왼쪽으로 곱해 내려가면, 다음 요소는 앞서 구한 미분값을 재사용할 수 있어 중복 계산이 없어지게된다. 그래서 역전파는 오른쪽 → 왼쪽 방향으로 진행된다.

- forward prop을 derivative를 구하는 방법으로 사용한다면, 파라미터가 p개 있는 경우 각 파라미터마다 그래프 전체(n개의 노드)를 처음부터 끝까지 계산해야한다 = n × p 단계
- back prop은 출력 J에서 거꾸로 한번 내려오면서, 각 노드의 미분 값을(모든 파라미터의 기울기값) 저장하고 재사용하기때문에 중복되는 미분연산이 없게 된다 = n + p 단계
