## 1. 진단 (Diagnostic)

학습 알고리즘에서 무엇이 잘 작동하고 무엇이 잘못되었는지 통찰력을 얻고, 성능 향상을 위한 올바른 방향성을 잡기 위해 실행하는 테스트를 의미.

---

## 2. 모델 평가 (Evaluating Model)

모델이 잘 동작하는지 알아보기위해, 모델 f(x)를 그래프로 그려서 데이터셋과 비교하여 평가할 수 있지만 특성이 1, 2개를 넘어가서 그래프로 그리기 어려운 경우에는 모델이 얼마나 잘하는 지 평가할 더 체계적인 방법이 필요하다.

→ 모든 데이터를 다 써서 파라미터를 학습 시키는 대신, 데이터셋을 두 개의 부분집합으로 나누는 기법 (훈련세트, 테스트 세트).

- 훈련세트로 파라미터(w, b)를 맞췄기 때문에, 학습에 쓰지 않은 테스트 세트를 따로 떼서 처음 보는 데이터에서의 성능을 확인하기 위한 방법
- 훈련세트로만 모델의 파라미터를 학습시키고, 그 다음 테스트 세트로 성능을 시험 (70-30 또는 80-20으로 나눈다)
- 테스트 세트의 오차값인 J_test는 모델이 새로운 데이터에 얼마나 잘 일반화 되는지를 가늠하게 해주는 추정치다

$$J_{train}(\vec{w}, b) = \frac{1}{2m_{train}} \left[ \sum_{i=1}^{m_{train}} \left(f_{\vec{w},b}(\vec{x}^{(i)}) - y^{(i)}\right)^2 \right]$$

$$J_{cv}(\vec{w}, b) = \frac{1}{2m_{cv}} \left[ \sum_{i=1}^{m_{cv}} \left(f_{\vec{w},b}(\vec{x}_{cv}^{(i)}) - y_{cv}^{(i)}\right)^2 \right]$$

$$J_{test}(\vec{w}, b) = \frac{1}{2m_{test}} \left[ \sum_{i=1}^{m_{test}} \left(f_{\vec{w},b}(\vec{x}_{test}^{(i)}) - y_{test}^{(i)}\right)^2 \right]$$

훈련 세트에서의 오차값을 구하고 테스트 세트에서의 오차값을 구한 후에 둘을 비교했을 때, 훈련세트의 오차값이 낮지만 테스트 세트의 오차값이 높다면 모델이 훈련 데이터에 과적합 되어 새로운 데이터들에 일반화하기는 좋지 않다는 것을 확인할 수 있다.

(평가용 오차에선 모델의 성능을 순수하게 측정하는게 목적이기 때문에 정규화항을 빼고 오차항만 계산한다.)

- 모델이 학습할 때 최소화하는 것 = 비용함수 = (오차항 + 정규화항)
- J_train, J_test로 평가할 때 재는 오차값 = 오차항만 (순수한 오차 측정)

### < 분류 문제에서의 오차 측정 >

Classification 문제에서는 오차값을 예측값과 실제값의 제곱 차이 또는 로지스틱 손실로 구하지않고, 더 직관적인 방식으로 예측값과 실제값이 같은지 다른지(일치 여부)만 따진다. (= 예측이 실제와 틀린 예제의 비율)

→ 모델을 학습시킬 때는 얼마나 심하게 틀렸는지까지 구분하기에 로지스틱 손실을 사용하지만, 모델의 평가 단계에선 사람이 이해하기 쉬운 지표인 오분류율을 사용한다.

계산 방법 (이진 분류 기준) — 먼저 각 예제를 0 또는 1로 확정 예측한다:

$$
\hat{y} =
\begin{cases}
1 & f(x) \geq 0.5 \\
0 & f(x) < 0.5
\end{cases}
$$

그다음 예측 $\hat{y}$ 가 실제 정답 y와 다른 예제가 전체에서 차지하는 비율을 센다:

$$J_{test} = \frac{\text{틀리게 분류한 테스트 예제 수}}{\text{전체 테스트 예제 수}}$$

$J_{train}$ 도 같은 방식으로 훈련 세트에서 계산.

측정 방식만 바뀌었을 뿐, J_train에서 틀린 비율과 J_test에서 틀린 비율을 비교해서 진단하는 흐름은 동일.

## 3. 모델 선택 (Model Selection)

앞서 테스트 오차로 모델 성능을 평가하는 방법을 배웠지만, 모델을 만들 땐 파라미터 w, b만 정하는 게 아니라 어떤 모델/하이퍼파라미터를 쓸지(몇 차 다항식 d, 정규화 λ 등)도 골라야 하기 때문에 모델 선택을 위한 검증 세트도 따로 만들어야 한다.

(모델 선택까지 테스트 세트로 진행하면 테스트 세트에 가장 잘 맞는 모델을 고른 셈이기에 테스트 세트가 오염된다.)

→ 데이터셋을 3개의 부분집합으로 나누는 기법 (훈련세트, 검증 세트, 테스트 세트).

Training set로 모든 후보 모델을 학습(데이터에 가장 잘 맞는 파라미터들을 학습)하고, Cross-validation set로 모델을 선택(데이터에 잘 맞는 다항식의 차수 또는 하이퍼 파라미터를 결정)한 후, Test set로 그렇게 정한 모델의 일반화 성능을 공정하게 확인 할 수 있다.

### < 결론 >

훈련 세트로 학습 → 검증 세트로 선택 → 테스트 세트로 최종 평가 절차는 다항식의 차수를 고를 때뿐만 아니라 신경망 구조(층 수, 유닛 수)를 고르는 경우등에도 똑같이 적용된다.

모델에 관한 결정을 내려야 한다면, 모든 결정을 오직 훈련세트 + 교차 검증세트만 사용해서 내리고 최종 모델을 정한 다음에만 어떤 결정에도 쓰이지 않아 오염되지 않은 테스트 세트로 평가할 수 있게 된다.

→ 모델 선택의 일반적 표준 절차다.

## 4. 편향과 분산 (Bias and Variance)

High Bias(Underfit)를 가지는 모델의 특징 중 하나는 훈련세트에서 모델이 잘 동작 하지 않는다는 특징이다 = 훈련 오차값 크다.

High Variance(Overfit)를 가지는 모델의 특징 중 하나는 검증 오차가 훈련 오차보다 훨씬 높다는 특징.

모델이 복잡해질수록(선형회귀에서 다항식의 차수가 증가), 훈련세트에서의 오차값은 줄어들게 된다. 모델이 아주 복잡하거나, 아주 단순한 경우에는 검증세트에서의 오차값은 커지게 되며 적당한 복잡도를 가질 때에 오차값이 낮아지게 된다.

high bias와 high variance가 동시에 나타나는 경우는 잘 일어나지 않지만, 드물게 존재한다 (= 일부 구간에선 과적합되지만 다른 일부 구간에선 과소적합인 형태).

### < 정규화 파라미터 lambda가 bias와 variance에 끼치는 영향 >

lambda 값이 매우 크다면, 알고리즘이 각 가중치들을 아주 작게 유지하기 때문에 각 가중치들이 0에 가까운 값으로 수렴하게 되고 결국 모델은 상수항 b와 유사한 직선의 형태를 가진다 → high bias = Underfit.

lambda 값이 매우 작거나 0이라면, 정규화(regularization) 과정이 없기 때문에 아주 복잡한 모델을 가지게 되기에 훈련 데이터에 과적합된 모델을 가지게 된다 → high variance = Overfit.

→ 적당한 값의 lambda를 선택해, 작은 훈련오차값과 작은 검증오차값을 가지게 해야 한다.

좋은 람다값을 고르는 방법은 교차 검증으로 모델의 복잡도를 고르던 것과 비슷한 절차로 구할 수 있다.

- lambda의 넓은 범위의 값들을 시도해서 각 정규화 값으로 파라미터를 맞춘 뒤, 검증 세트에서의 오차값을 비교해서 평가하면 정규화 파라미터인 lambda의 가장 좋은 값을 고를 수 있다.

## 5. 성능의 기준선 (Baseline level of performance)

기준선 성능: 학습 알고리즘이 궁극적으로 도달하기를 기대하는 합리적인 오차 수준.

훈련 오차가 높은지 판단하기 위해서, 기준선 성능을 설정하는 것이 유용하며 기준선 성능을 설정하는 대표적인 방법으로는:

- 사람이 해당 작업을 얼마나 잘하는지 측정해서 사용
- 경쟁 알고리즘이나 기존 시스템의 성능을 측정해서 사용
- 이전의 경험을 기반으로 측정해서 사용

성능의 기준선은 어떤 작업인지에 따라 달라진다. 완벽한 성능이 가능한 작업이면 기준선은 0%, 음성 인식처럼 오디오 자체가 시끄러워서 사람조차 완벽할 수 없는 작업이라면 기준선은 0%가 아닌 더 높은 값.

성능의 기준선을 정한 후엔, 첫째로 훈련 오차와 기준선 사이의 차이를 비교하고 둘째로 훈련 오차와 검증 오차 사이의 간격을 비교한다.

- 훈련 오차와 기준선 사이의 차이가 크다면, high bias 문제
- 훈련 오차와 검증 오차 사이의 차이가 크다면, high variance 문제

→ 편향 / 분산 진단은 절대값이 아닌 기준선, 훈련 오차, 검증 오차간의 간격의 크기로 판단한다.

## 6. 학습 곡선 (Learning Curve)

훈련 데이터가 늘어날 때 훈련 오차값과 검증 오차값이 어떻게 변하는지 그린 그래프.

training data set이 커질수록, 검증오차값은 감소하며 훈련 오차값은 증가하게 되고
→ 훈련 데이터가 많아질수록 모든 데이터에 완벽히 맞추기 힘들어지기 때문에.

training data set이 작을수록, 검증 오차값은 커지며 훈련 오차값은 감소하게 된다.
→ 훈련 데이터가 적기에, 모든 데이터에 완벽히 맞추기 쉬워진다.

(검증 오차값은 일반적으로 훈련 오차값보다 큰 값을 가진다. 파라미터를 훈련 세트에 학습 시켜 맞췄기 때문.)

### < High bias인 경우 >

모델이 high bias를 가지는 경우에는 훈련 데이터의 크기를 키우는 것은 별로 도움이 되지 않는다.
→ 모델이 너무 단순하기에 훈련 데이터를 더 많이 준다고 해서 바뀌는 게 없기 때문에 모델을 더 복잡하게 만들어야 한다.

(학습 곡선에서 $J_{train}$ 과 $J_{cv}$ 가 둘 다 human level performance 기준선보다 한참 위에서 평평해진다.)

### < High variance인 경우 >

반대로 모델이 high variance를 가지는 경우에는 훈련 데이터의 크기를 키우는 것이 도움이 된다.
→ 데이터가 많아질수록, 모델이 복잡해도 모든 데이터를 완벽히 외울 수 없어지기 때문에 데이터의 전체적인 패턴을 따라가기 때문에 훈련 데이터를 늘리는 것이 도움이 된다.

(학습 곡선에서 $J_{train}$ 과 $J_{cv}$ 사이의 간격이 크고, 데이터가 늘어날수록 그 간격이 좁혀진다.)

## 7. 학습 알고리즘 디버깅 (Debugging a learning algorithm)

모델을 설계한 후에, prediction에서 허용할 수 없는 수준의 오차가 발생하는 경우 취할 수 있는 방법들:

| 방법 | 해결하는 문제 |
|:---|:---|
| Get more training examples | fixes high variance (과적합 방지) |
| Try smaller sets of features | fixes high variance (과적합 방지) |
| Try getting additional features | fixes high bias (과소적합 방지) |
| Try adding polynomial features | fixes high bias (과소적합 방지) |
| Try decreasing lambda | fixes high bias (과소적합 방지) |
| Try increasing lambda | fixes high variance (과적합 방지) |

## 8. 신경망과 편향/분산 (Neural networks and bias variance)

머신러닝에선 모델이 너무 단순하면 높은 편향, 너무 복잡하면 높은 분산을 가지기에 이 두 상태 사이에서의 균형을 찾아야 최선의 결과를 얻을 수 있었지만 신경망은 편향-분산 tradeoff에서 벗어나는 방법을 제공한다.

(물론 머신러닝도 모델의 복잡도를 높여서 높은 편향 문제를 해결하고, 높은 분산 문제를 정규화와 데이터 추가로 해결할 수 있지만, 머신러닝에선 차수를 너무 높이면 수치적으로 불안정해지고 다루기 힘들어지는 반면에 신경망은 층/유닛을 계속 늘려도 안정적으로 학습한다. 또한 신경망은 어떤 복잡도가(어떤 차수를 써야할지 등) 좋은지 고르지 않고 그냥 크기만 계속 키우면 되기에 편리하다.)

신경망은 크기를 충분히 크게 만들수록 데이터가 지나치게 거대하지만 않다면 거의 항상 훈련 데이터를 잘 맞출 수 있기에 편향 문제가 해결된다. (단, 커지는 계산 비용문제와 데이터가 너무 거대하면, 아무리 큰 신경망도 완벽히 못 맞출 수 있다는 한계 존재.)

→ 먼저 J_train(= 훈련 오차값)을 보고 해당 값이 크면 신경망 크기를 크게 만듬으로써 모델이 높은 편향을 가지는 것을 방지하고, 그 다음 J_cv(= 검증 오차값)를 보고 값이 크면 데이터를 추가함으로써 모델이 높은 분산을 가지는 것을 방지. 이 과정을 편향, 분산이 둘 다 낮아질 때까지 반복한다.

→ 신경망의 크기가 커지면 과적합 위험이 커지지만, 적절한 정규화를 거치면 과적합(높은 분산) 위험을 억제해줘서, 결과적으로 크기가 작은 신경망만큼 잘하거나 오히려 더 잘한다. 크기를 키우는 대가는 오로지 계산 비용이 증가해서 시간과 자원이 더 든다는 점.

## 9. 오류 분석 (Error Analysis)

오류 분석: 학습 알고리즘이 잘못 분류하거나 잘못 예측한 예제들을 수동으로 직접 살펴보는 것.

틀린 예제들을 직접 하나하나 살펴봄으로써 틀린 예제들의 패턴을 파악할 수 있고, 다음에 무엇을 하면 좋을지와 관련된 정보를 알 수 있다 (추가로 무엇을 안해도 되는지도 알려준다 - 아주 드문 오류 유형의 경우 가치가 떨어진다).

오류 분석의 한가지 한계는, 사람이 잘하지 못하는 문제에서는 하기가 훨씬 어렵다는 점 → 사람이 정답과 그 이유를 알 수 있는 작업에서만 효율적이다.

→ 오류 분석은 실수의 패턴을 파악해서 어떤 특징을 추가할지, 무엇에 집중할지 등 세부적인 요소들을 결정한다 (편향/분산은 모델의 큰 방향을 결정 - 데이터의 추가 또는 모델의 복잡도 등).

## 10. 데이터 중심 접근: 데이터로 성능 높이는 방법

단순히 새로운 데이터를 추가할때 모든 종류의 데이터를 추가하는 것이 아니라 오류 분석을 통해 파악한 패턴으로 효율적으로 모델의 성능을 올릴 수 있는 특정 유형의 데이터만 추가하는 방법 (표적 수집).

### < 데이터 증강 (Data augmentation) >

이미지나 음성 데이터 같은 경우, 아예 새로운 데이터를 추가하는 것이 아닌 data augmentation를 통해 기존의 데이터셋을 활용해 새로운 데이터를 만드는 방법.

Data augmentation (데이터 증강): 기존의 데이터를 변형(회전, 축소, 확대 등)시켜서 똑같은 정답(레이블)을 가지는 새로운 데이터를 만드는 기법. → 모델이 더 다양한 형태를 인식하게 되서 견고해진다.

데이터 증강의 유의점: 데이터에 가하는 변화나 왜곡이 test set에 있을법한 변화나 왜곡의 유형을 대표해야한다. 의미 없는 변화나 왜곡을 추가하면 실제로 도움이 별로 되지 않는다. = 테스트 세트를 닮은 변형.

### < 데이터 합성 (Data synthesis) >

완전히 처음부터 새로운 예제를 만들어내는 기법. 주로 사진 OCR과 같은 이미지 작업에서 많이 사용한다.

→ 머신러닝 성능을 높이는 데는 모델 중심(코드, 알고리즘 개선)과 데이터 중심(데이터 개선) 접근 방식이 있으며, 데이터를 잘 설계하는 데이터 중심 접근이 모델의 성능 개선에 더 효율적일 때가 많다.

## 11. 전이학습 (Transfer Learning)

전이학습: 하나의 대규모 데이터 세트로 사전학습된 신경망의 파라미터를, 입력 데이터 타입이 같은 다른 목적의 데이터셋에 초기값 또는 고정된 파라미터로 재사용해서 학습 효율과 성능 높이는 기법. 2단계를 거친다.

1. 지도 사전 학습(Supervised pre-training): 신경망을 우리의 최종 목표와는 완전히 일치하지는 않는 아주 거대한 데이터 세트로 훈련시키는 단계
2. 미세 조정(Fine-tuning): 지도 사전 학습을 통해 얻거나 초기화된 파라미터를 가져와서, 경사 하강법을 추가로 실행하여 손글씨 숫자 인식이라는 특정 목적에 맞게 가중치를 정밀하게 조정하는 과정
   - 옵션 1: 출력층만 학습, 나머지 층들은 사전 훈련된 파라미터를 그대로 고정해서 이용 → 데이터 적을 때 효율적
   - 옵션 2: 사전 훈련된 파라미터를 초기값 삼아 출력층을 포함한 전체 층 재학습 → 데이터 충분할 때 효율적

전이학습이 가능하고 효과적인 이유: 신경망은 층이 깊어질수록 (edge → corner → shape, curve)와 같이 점점 더 복잡한 패턴을 학습하기 때문에 완전히 다른 인식 작업이라도 입력의 데이터 타입이 같다면 해당 데이터 타입에 공통으로 적용되는 일반적인 특징들을 학습하기에 좋은 성능을 낼 수 있다.

## 12. 불균형 데이터를 위한 오차 지표 (Error metrics for skewed data)

- Precision: Positive라고 모델이 예측한 것 중에서 실제로 True Positive인 확률
- Recall: 실제로 Positive 인 것들 중에서, 모델이 Positive라고 올바르게 예측한 것의 확률

불균형한 데이터셋이 있는 경우(희귀한 class를 분류하는 경우)에, 정확도만으로 알고리즘의 좋고 나쁨을 판단할 수 없다. 정확도는 전체 데이터에서 맞춘 비율만 계산하기에 모델이 다수 클래스만 예측해도 소수 클래스를 전혀 맞추지 못한 채 높은 값이 나올 수 있기 때문에, class별 성능을 구분해서 보여주지 못한다.

→ 그렇기에 precision과 recall을 두 값 모두 높은 값을 나타내는지 확인함으로써 학습 알고리즘이 유용한지 확인할 수 있다.

### < Precision과 Recall의 Trade off >

모델이 출력하는 확률값(0~1)을 얼마부터 공격(1)이라고 판단할 건지 기준선인 Threshold(임계값)을 어떻게 정하는지에 따라 precision과 recall값은 달라진다.

- Threshold 값을 높이면 더욱 보수적으로 판단하면서 확실한것만 판단하기에 precision은 높아지지만, 반대로 애매한 것들은 놓쳐버리기에 recall이 떨어진다.
- Threshold 값을 낮추면 더욱 공격적으로 판단하면서 의심스러운 모든 것들을 잡아내기에 recall은 높아지지만, 반대로 조금만 이상해도 다 잡아내기에 precision이 떨어진다.

→ 상황과 목적에 따라 다르게 설정한다. 오탐의 비용이 높다면 Threshold 값을 높게 설정하고 미탐의 비용이 높다면 Threshold값을 낮게 설정한다.

### < F1 score >

precision과 recall을 단일 점수로 결합한 점수. 둘 중 더 낮은 값에 더 많은 가중치를 부여한다.

→ 극단적으로 한쪽 값만 튀는 것에 페널티를 주고, 두 값이 균형 있게 높을 때만 높은 점수를 준다.
→ 정밀도와 재현율의 조화 평균.

$$F_1 = \frac{1}{\frac{1}{2}\left(\frac{1}{P} + \frac{1}{R}\right)} = 2 \frac{PR}{P + R}$$
